# SimpleBTC 使用指南

## 🚀 快速开始

### 选项 1: 命令行演示（最简单）

```bash
cargo run --bin btc-demo --release
```

这将运行一个完整的演示，展示所有功能。

### 选项 2: Web界面 + API（推荐用于视频录制）

**步骤 1: 启动API服务器**

```bash
cargo run --bin btc-server --release
```

你会看到：
```
========================================
   SimpleBTC API Server
========================================
🚀 服务器运行在: http://127.0.0.1:3000
```

**步骤 2: 启动Electron GUI（需要Node.js）**

在新终端：

```bash
cd frontend
npm install    # 首次运行需要安装依赖
npm start
```

Electron窗口会自动打开！

**步骤 3: 运行演示**

在Electron界面中：
1. 点击右上角的 **"🎬 运行演示"** 按钮
2. 观看自动化演示过程
3. 查看区块链实时更新

## 🎨 界面功能说明

### 左侧面板

**💰 钱包管理**
- 点击"创建新钱包"按钮创建
- 显示所有钱包地址和余额
- 点击钱包自动填充到发送地址

**💸 发起转账**
- 发送地址: 转账发起者
- 接收地址: 接收方
- 金额: BTC数量
- 手续费: 越高越优先被打包

**⛏️ 挖矿**
- 输入矿工地址
- 点击"开始挖矿"
- 矿工获得: 基础奖励(50) + 所有交易手续费

### 右侧面板

**⛓️ 区块链浏览**
- 实时显示所有区块
- 最新区块在顶部
- 点击查看交易详情
- 显示手续费信息

### 顶部信息栏

- **区块高度**: 当前区块数量
- **难度**: 挖矿难度（默认3）
- **待处理交易**: 交易池中的交易数

## 🔧 API使用示例

### 创建钱包

```bash
curl -X POST http://127.0.0.1:3000/api/wallet/create
```

返回:
```json
{
  "success": true,
  "data": {
    "address": "9da1db58fa9fcb4651afa0b84c8e05ac89fc7f78",
    "public_key": "..."
  }
}
```

### 查询余额

```bash
curl http://127.0.0.1:3000/api/wallet/balance/9da1db58fa9fcb4651afa0b84c8e05ac89fc7f78
```

### 创建交易

```bash
curl -X POST http://127.0.0.1:3000/api/transaction/create \
  -H "Content-Type: application/json" \
  -d '{
    "from_address": "9da1db58fa9fcb4651afa0b84c8e05ac89fc7f78",
    "to_address": "ed3de77bc62ad1a884e57b1d9c19571169c967b3",
    "amount": 30,
    "fee": 5
  }'
```

### 挖矿

```bash
curl -X POST http://127.0.0.1:3000/api/mine \
  -H "Content-Type: application/json" \
  -d '{"miner_address": "9da1db58fa9fcb4651afa0b84c8e05ac89fc7f78"}'
```

### 验证区块链

```bash
curl http://127.0.0.1:3000/api/blockchain/validate
```

## 📊 性能特性

### 交易索引

系统自动为每个区块建立索引，支持：
- 快速查找交易所在区块
- 按地址查询所有相关交易
- O(1) 复杂度查询

### 批量处理

支持批量验证交易，提升性能

### 交易池优化

自动按费率排序，矿工优先打包高手续费交易

## 🐛 常见问题

### Q: API无法连接？
A: 确保API服务器正在运行（cargo run --bin btc-server --release）

### Q: Electron无法启动？
A: 确保已安装Node.js，并运行 `npm install`

### Q: 挖矿失败"没有待处理的交易"？
A: 正常现象，只有当交易池有交易时才能挖矿

### Q: 交易被拒绝"余额不足"？
A: 检查发送地址是否有足够余额，记得包含手续费

## 📝 注意事项

1. **这是演示项目**: 密钥管理简化，不用于生产
2. **数据不持久**: 重启服务器会重置区块链
3. **单机运行**: 没有P2P网络
4. **简化签名**: 实际BTC使用ECDSA
